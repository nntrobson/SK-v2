Pipeline Overview

Each shot's trajectory is a sequence of (x, y) coordinates in inches, measured from the camera crosshair. The coordinates are globally stabilized (camera motion removed via optical flow) and position-normalized (station perspective correction applied).

The pipeline classifies each shot's direction in 5 steps:

1 Origin Normalization

Translate each trajectory so the first detection is at (0, 0). This makes all trajectories comparable regardless of where the clay first appeared on screen.

x'[i] = x[i] - x[0]
y'[i] = y[i] - y[0]

2 Endpoint Trimming

Remove the first and last trajectory point (if > 4 points total). These are the noisiest detections:

  • First point: clay is smallest/farthest, worst bounding box
  • Last point: may be post-break debris or tracking drift at edge of frame

3 Outlier Rejection

Two criteria flag a point as an outlier:

A) Backward motion: If a point moves backward along the overall 2D trajectory direction (first→last), it's likely a tracking glitch or detection bounce. The check uses the dot product of each step vector with the dominant direction unit vector.

dir = normalize(point[last] - point[first])
step[i] = point[i] - point[i-1]
projection = dot(step[i], dir)
outlier if projection < 0 (point moves against dominant direction)

B) Mahalanobis distance: Compute the step vector (dx, dy) between consecutive clean points. Fit a 2D Gaussian to the step distribution. Any step > 3 Mahalanobis standard deviations from the mean is an outlier. This catches sudden jumps in any direction while being robust to the natural shape of the trajectory.

step[i] = (x[i] - x[i-1], y[i] - y[i-1])
μ = mean(steps), Σ = covariance(steps)
D_mah = sqrt((step - μ)^T * Σ^-1 * (step - μ))
outlier if D_mah > 3

This removes: tracking glitches, post-break debris, stabilization artifacts, and detector jumps.

4 Parabolic Fit (Projectile Model)

Fit x = ay^2 + by + c via least-squares. This models projectile motion where gravity curves the trajectory while the clay drifts laterally.

  • a = curvature (gravity + wind effect on horizontal position)
  • b = initial lateral velocity (launch direction)
  • c = x-offset at origin (near 0 after normalization)

Direction is the tangent slope at the midpoint of the y-range:

dx/dy at y_mid = 2a * y_mid + b
angle = atan(dx/dy) * 180 / pi

The midpoint tangent equals the average tangent over the entire trajectory, giving the dominant direction during flight rather than being skewed by launch or terminal phases.

5 Angle-Based Classification

Classify based on the angle from vertical (0° = straight up):

|angle| < moderate_threshold → straight
moderate_threshold ≤ |angle| < hard_threshold → moderate_left / moderate_right
|angle| ≥ hard_threshold → hard_left / hard_right
Sign: negative = left, positive = right

Thresholds are auto-detected from natural breaks (largest gaps) in the angle distribution, then adjustable via sliders.

Quality Indicator: R-squared

R^2 measures how well the parabola fits the data. Values above 0.9 indicate a clean trajectory. Low R^2 (< 0.7) suggests tracking problems or the clay broke mid-flight. The fit quality is shown per-shot in the detail view.
